Common Frontend Errors to Check

When testing the main app (http://localhost:3000), open your browser's developer console (F12 or Right-click -> Inspect -> Console tab) and look for these common errors:

  • Red Error Messages: Any red text indicates a JavaScript runtime error. Copy the full message and stack trace.
  • 404 Not Found: If you see 404 errors for images or API calls, check the URLs and ensure the backend services are running.
  • CORS Errors: "Cross-Origin Request Blocked" or similar messages mean the frontend isn't allowed to talk to the backend.
  • Hydration Errors: "Text content did not match" or "Prop `className` did not match" usually indicate server-side rendering (SSR) issues.
  • Image Loading Errors: "Invalid src prop" for `next/image` means the image domain is not configured in `next.config.js`.
  • Uncaught (in promise) errors: Often related to failed API calls or unhandled asynchronous operations.

If you encounter any of these, please copy the full error message and share it. This helps in debugging!
